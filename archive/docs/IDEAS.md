# Ideas and Open Questions

## Core Ideas
- A mining-focused automation engine
- Support for reusable profiles
- Structured action execution
- Visual-state-driven reactions
- Clear separation between engine and configuration

## Product Shape Questions
- Should the app feel more like a runner or more like a configurable editor?
- Should profiles be simple JSON definitions first?
- Should the first UI be minimal or moderately interactive?
- Should execution begin manually only, or support saved auto-start behavior later?

## Technical Questions
- What visual detection method should be prioritized first?
- How should internal state be represented?
- How should timing, retries, and safety stops be handled?
- What level of logging is necessary from v1?
- Should capture/detection be region-based from the start?

## Future Notes
This file is intentionally informal and should collect unresolved thoughts until the roadmap is ready.
